把你坑哭的Flexible SSL死循环。

面向中国 / 香港运维的 Cloudflare CLI。一条命令定位
你的站为什么在转到 Cloudflare 之后坏掉了。

30 秒上手 GitHub
install + cfcn ssl diag 
$ curl -fsSL https://raw.githubusercontent.com/tianmind-studio/cloudflare-cn-kit/main/install.sh | bash
$ export CFCN_TOKEN="***"
$ cfcn ssl diag example.com

==> SSL diagnosis: example.com
    zone:      example.com
    ssl mode:  flexible
    proxied:   true
    probing edge and origin...
    Edge HTTP/80:       301
    Edge HTTPS/443:     301
    Edge HTTPS with -L: 000  (up to 10 redirects)
    origin IP:          203.0.113.10 (DNS A)
    Origin HTTP/80:     301
    Origin HTTPS/443:   000

==> Diagnosis
warn: SSL mode is 'flexible' AND DNS is proxied.
→ This is the classic Flexible-SSL redirect loop setup.
→   - CF talks to origin over HTTP, but origin (nginx)
→     redirects HTTP to HTTPS. CF passes the 301, client loops.
warn: direct origin HTTP redirects, but direct origin HTTPS is unreachable.
→ Likely origin has no valid cert installed while nginx force-redirects to HTTPS.

为什么这是核心

迁站到 Cloudflare 后坏掉的 #1 原因。全中文运维社区搜"Cloudflare 重定向循环"都是这个。

◆ 症状

浏览器 ERR_TOO_MANY_REDIRECTS · curl -I 一遍遍返回 301 · 有时候中国 ISP 会直接给你假装成 403 / 502。

◆ 根因

CF Flexible 模式下，CF ↔ 源站走 HTTP。如果 nginx 写了 return 301 https，CF 就把 301 透传回浏览器，循环触发。

◆ 修法

两条干净路径：A) 改成 Full(strict) + 源站装证书 · B) 源站去掉 HTTPS 跳转，让 CF "Always Use HTTPS" 做。完整分析。

其它命令

日常 CF 运维该用的都在这里。

⬢ zone

list / show / ssl / purge — 读你全部 zone 的 SSL 模式，切模式，清缓存。

⬢ dns

add / del / list / bulk / wildcard / export — FQDN 自动推断 zone，YAML 批量，多租户泛解析一条命令。

⬢ ssl

diag / mode / hsts — 除了 diag，还有 HSTS 切换。注意：HSTS 一开难撤。

⬢ cn

ping / trace — 把你指向 itdog.cn / chinaz.com / ipip.net，不是你 laptop 上的 ping。

⬢ doctor

验证 token scope。放心跑 destructive 命令之前先来一发。

⬢ --dry-run · --json

任何写操作都可预览。任何读操作都能出原始 JSON 喂给 jq。
